# 数据格式说明

## 输出结构

每个城市爬取完成后，在 `output/` 目录下生成对应的 CSV 文件。

### 文件名格式

```
{城市名}_baidu_index.csv
```

例如：`上海_baidu_index.csv`、`北京_baidu_index.csv`

### CSV 字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `date` | 字符串 | 日期，格式 YYYY-MM-DD | `2026-03-01` |
| `keyword` | 字符串 | 关键词 | `人工智能` |
| `city` | 字符串 | 城市名称 | `上海` |
| `type` | 字符串 | 指数类型：`all`/`pc`/`wise` | `all` |
| `index` | 数字（字符串） | 百度指数数值 | `372` |

### type 说明

| type | 含义 | 说明 |
|------|------|------|
| `all` | 综合搜索指数 | PC端 + 移动端的总和 |
| `pc` | PC端搜索指数 | 仅在电脑上的搜索量 |
| `wise` | 移动端搜索指数 | 仅在手机上的搜索量 |

## 数据示例

```csv
date,keyword,city,type,index
2026-03-01,人工智能,上海,all,372
2026-03-01,大数据,上海,all,372
2026-03-01,云计算,上海,all,372
2026-03-01,人工智能,上海,pc,152
2026-03-01,大数据,上海,pc,152
2026-03-01,云计算,上海,pc,152
2026-03-01,人工智能,上海,wise,220
2026-03-01,大数据,上海,wise,220
2026-03-01,云计算,上海,wise,220
```

## 计算规则

```
每个城市数据量 ≈ 关键词数量 × 天数 × 3（type）
示例：52关键词 × 15天 × 3type = 2340 行
```

**关于同批关键词值相同**：
百度指数 API 对同一批次请求（最多3个关键词）返回相同的指数序列。这意味着如果你在一批中请求了"十五五规划"、"高质量发展"、"经济增长"三个关键词，它们在同一天的指数值是相同的。

## 数据可靠性说明

- **指数为 0**：表示该关键词在该时间范围没有搜索数据，或百度未收录该词
- **部分缺失**：某些关键词可能只返回部分日期的数据
- **数据准确度**：数据直接来自百度指数 API，与在 https://index.baidu.com 网页上看到的一致

## 与 v1（qdata 版）的兼容性

| 版本 | CSV格式 | 扩展名 | 编码 |
|------|---------|--------|------|
| v1 (qdata) | date,keyword,city,type,index | .csv | utf-8-sig |
| v2 (Scrapling) | date,keyword,city,type,index | .csv | utf-8-sig |

**完全兼容**，可以直接替换使用，无需修改后续分析流程。
